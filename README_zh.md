# 📂 文件管理系统 (File Manager System) —— 你的海量资源“大管家”

<div align="center">

[English](README.md) | [中文]

</div>

这是一个**为 NAS 而生**、轻量级、高性能的文件管理系统，专门用来对付成千上万的乱糟糟的文件。它不仅能帮你把文件整理得井井有条，还能通过“黑科技”帮你找出那些占地方的重复图片和视频。

---

## 🚀 1分钟快速启动 (Docker)

想直接体验？只需一行命令（记得把 `xxx` 换成你电脑里存文件的文件夹路径）：

```bash
docker run -d \
  --name file-manager-system \
  -p 8080:8080 \
  -v ${pwd}/data:/app/data \
  -v xxx:/file_repository \
  ghcr.io/xie87241203/file_manager_system:latest
```

---

## ✨ 它能为你做什么？

*   **构建索引**: 通过扫描仓库构建文件索引，方便快速搜索。
*   **预览媒体**: 生成缩略图后能够预览图片和视频。
*   **文件查重**: 通过MD5和感知哈希 (pHash)识别相同文件和内容相似的视频/图片为你找出重复文件

---

## 📖 怎么开始玩？

1.  **启动并登录**: 容器跑起来后，在浏览器输入地址进入。默认账号为 `admin`，默认密码为 `admin123`。
2.  **安全第一**: 登录后，请务必通过系统设置修改你的用户名和密码。
3.  **扫一扫**: 在“系统设置”里填好路径，然后去“文件仓库”点一下扫描。
4.  **造预览**: 点击下方的“生成缩略图”，系统就会在后台默默为你准备预览图。
5.  **去重**: 感觉硬盘快满了？跑一遍“文件去重”，把那些没用的“分身”全删了。

---

## ⚙️ 常见配置 (映射关系)

| 容器内路径 | 它是干嘛的 | 建议 |
| :--- | :--- | :--- |
| `/app/data` | 存数据库和缓存 | 必须映射，否则重启后数据会丢 |
| `/file_repository` | 你的文件库 | 映射到你想管理文件的那个文件夹 |
